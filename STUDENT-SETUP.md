# מדריך התקנה לתלמידים — חבילת הסקילים של הסטודיו (VS Code)

בסוף המדריך הזה יהיו לכם 6 סקילים מקצועיים שגורמים ל-Claude לבנות בקאנד, מסדי נתונים, אבטחה, טסטים, משחקים ודיפלוי — ברמה מקצועית, אוטומטית, בלי שתצטרכו לבקש.

זמן משוער: 10 דקות. עושים את זה **פעם אחת** — הסקילים יעבדו אחר כך בכל פרויקט.

---

## שלב 0 — מה צריך שיהיה מותקן מראש

1. **VS Code** — [code.visualstudio.com](https://code.visualstudio.com)
2. **Claude Code** — מתקינים אחת משתי הדרכים:
   - **תוסף VS Code (הכי פשוט):** ב-VS Code → Extensions (Ctrl+Shift+X) → חפשו "Claude Code" → Install
   - **או דרך הטרמינל:** `npm install -g @anthropic-ai/claude-code` (דורש Node.js)
3. **חשבון Claude** — בהפעלה הראשונה Claude Code יבקש להתחבר; עוקבים אחרי ההוראות בדפדפן.

בדיקה שהכול תקין: פתחו את Claude Code (ראו שלב 1) וכתבו לו "שלום". אם ענה — ממשיכים.

---

## שלב 1 — פותחים את Claude Code בתוך VS Code

יש שתי דרכים, שתיהן טובות:

- **דרך התוסף:** לחצו על אייקון Claude בסרגל הצד של VS Code — נפתח חלון צ'אט.
- **דרך הטרמינל המשולב:** Terminal → New Terminal (או `` Ctrl+` ``), ואז מקלידים:
  ```
  claude
  ```
  נפתח צ'אט של Claude בתוך הטרמינל.

> ⚠️ **הטעות הנפוצה ביותר:** את הפקודות של השלבים הבאים מקלידים **בתוך הצ'אט של Claude** (איפה שכותבים לו הודעות) — **לא** בטרמינל הרגיל של Windows/Mac. פקודה שמתחילה ב-`/` היא "פקודת סלאש" של Claude Code, והיא קיימת רק בתוך הצ'אט. אם הקלדתם אותה בטרמינל רגיל וקיבלתם שגיאה — זו הסיבה.

---

## שלב 2 — מוסיפים את "החנות" של הסטודיו

בתוך הצ'אט של Claude הקלידו (או הדביקו) ולחצו Enter:

```
/plugin marketplace add meylon7/claude-dev-skills
```

**מה הפקודה הזאת עושה?** תחשבו על זה כמו הוספת חנות אפליקציות לטלפון. עדיין לא התקנתם כלום — רק אמרתם ל-Claude Code: "יש קטלוג תוספים בכתובת הזאת ב-GitHub (המשתמש meylon7, הריפו claude-dev-skills) — תכיר אותו." Claude Code ניגש ל-GitHub, מוריד את הקטלוג, ורושם אותו אצלכם בשם **meylon-studio**.

אם הצליח — תראו הודעת אישור שהמרקטפלייס נוסף.

---

## שלב 3 — מתקינים את חבילת הסקילים

עכשיו, כשהחנות מוכרת, מתקינים ממנה את החבילה:

```
/plugin install claude-dev-skills@meylon-studio
```

**מה הפקודה הזאת עושה?** קוראים אותה כך: "התקן את התוסף ששמו `claude-dev-skills` **מתוך** המרקטפלייס ששמו `meylon-studio`". הפורמט `שם-תוסף@שם-חנות` קיים כי אפשר להוסיף כמה חנויות, ושתי חנויות יכולות להכיל תוספים באותו שם — ה-`@` אומר מאיפה בדיוק לקחת.

Claude Code יוריד את החבילה ויתקין את 6 הסקילים. ייתכן שיבקש אישור — אשרו.

---

## שלב 4 — בודקים שזה עובד

1. סגרו את הצ'אט ופתחו סשן חדש (בטרמינל: `claude` מחדש).
2. בקשו מ-Claude משהו כמו:
   ```
   תבנה לי endpoint שמקבל טופס הרשמה ושומר אותו
   ```
3. **איך יודעים שהסקילים פעילים?** Claude יתנהג אחרת: הוא יתחיל ממידול הנתונים, יוסיף ולידציה עם zod, ידבר על הרשאות ואבטחה — בלי שביקשתם. אפשר גם לשאול אותו ישירות: "אילו סקילים זמינים לך?" — הוא אמור למנות בין השאר את backend-api-design, app-security, database-design, testing-strategy, web-game-dev, ship-and-deploy.

---

## מה בעצם התקנתם? (למה זה שווה)

| סקיל | מה הוא גורם ל-Claude לעשות |
|---|---|
| `backend-api-design` | בקאנד בשכבות, ולידציה בכל קלט, טיפול שגיאות אחיד |
| `app-security` | אבטחה אוטומטית: הרשאות, סודות, הגנה מהזרקות |
| `database-design` | מידול נתונים לפני קוד, constraints, מיגרציות |
| `testing-strategy` | טסטים כחלק מההגדרה של "מוכן", לא כמחשבה שנייה |
| `web-game-dev` | משחקים עם game loop נכון ותחושה מקצועית |
| `ship-and-deploy` | העלאה לאוויר מסודרת: סביבות, CI, צ'קליסט השקה |

| בעיה | פתרון |
|---|---|
| `command not found: /plugin` או שהפקודה נשלחה כהודעה רגילה | הקלדתם בטרמינל רגיל במקום בצ'אט של Claude, או שגרסת Claude Code ישנה — עדכנו (`claude update` או עדכון התוסף) |
| שגיאת רשת ב-marketplace add | בדקו חיבור לאינטרנט; אם יש proxy ארגוני — נסו מרשת אחרת |
| ההתקנה עברה אבל Claude "לא מכיר" את הסקילים | פתחו סשן **חדש** — סקילים נטענים בתחילת סשן |
| שום דבר לא עובד | דרך חלופית למטה 👇 |

### דרך חלופית — התקנה ידנית (בלי plugin)

בטרמינל **רגיל** (כן, הפעם דווקא הרגיל 🙂):

```bash
git clone https://github.com/meylon7/claude-dev-skills
cd claude-dev-skills
# Mac/Linux/Git-Bash:
mkdir -p ~/.claude/skills && cp -r skills/* ~/.claude/skills/
# Windows PowerShell:
# New-Item -ItemType Directory -Force "$HOME\.claude\skills"; Copy-Item -Recurse -Force skills\* "$HOME\.claude\skills\"
```

התוצאה זהה — הסקילים פעילים בכל פרויקט. ההבדל היחיד: עדכונים עתידיים תצטרכו למשוך ידנית, בעוד שבהתקנת plugin מתעדכנים דרך `/plugin`.
